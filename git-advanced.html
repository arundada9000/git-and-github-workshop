Advanced Git features

Interactive rebase, cherry‑pick, bisect, submodules, and more

1. Interactive Rebase

Rewrite commit history by squashing, reordering, or editing commits. Use with caution (never on public branches).

git rebase -i HEAD~3   # modify last 3 commits

In the editor, you can pick, reword, edit, squash, fixup, or drop commits.

2. Cherry‑pick

Apply a specific commit from one branch onto another.

git cherry-pick a1b2c3d   # apply commit a1b2c3d to current branch

Useful for hotfixes or picking selective changes.

3. Git Bisect – binary search for bugs

Find which commit introduced a bug by automated bisection.

git bisect start
git bisect bad            # current commit is bad
git bisect good known-good-commit
# Git checks out a midpoint; you test and mark:
git bisect good  # or git bisect bad
# Repeat until the faulty commit is identified.
git bisect reset

4. Reflog – safety net for lost commits

Records every movement of HEAD and branch tips. Recover lost commits or resets.

git reflog
git reset --hard HEAD@{2}   # go back to where HEAD was two moves ago

5. Submodules – embedding external repositories

Include another Git repo as a subdirectory.

git submodule add https://github.com/user/lib.git lib
git submodule update --init --recursive   # after cloning

Submodules are pinned to a specific commit.

6. Worktrees – multiple working directories

Check out multiple branches simultaneously in separate folders, sharing the same .git.

git worktree add ../hotfix hotfix-branch

7. Git Hooks – custom scripts

Scripts in .git/hooks/ that run on events like pre‑commit, pre‑push, post‑merge. Example: a pre‑commit hook to run linters.

8. Advanced merging strategies

git merge --squash combines all commits from a branch into one. git merge --no-ff forces a merge commit even if fast‑forward is possible.

9. Practice questions

1. You want to combine the last four commits into one clean commit before pushing. What command?

Answer
git rebase -i HEAD~4
then mark commits as squash or fixup.

2. You need to apply a bugfix commit (hash f00d1e) from the main branch into your feature branch. How?

Answer
git cherry-pick f00d1e

3. You suspect a bug was introduced sometime in the last 50 commits. How can Git help you locate it?

AnswerUse git bisect to binary search the history.

4. After a mistaken git reset --hard, you lost your last commit. How can you recover it?

AnswerUse git reflog to find the commit hash and reset to it.
Advanced Git gives you surgical control over your history – use wisely.
