Git branches: master the branching

From basics to merge conflicts and workflows

1. What is a branch?

A branch in Git is a lightweight movable pointer to a commit. The default branch is usually called main (or master). Branches allow you to develop features, fix bugs, or experiment in isolation without affecting the main codebase.

2. Basic branch operations

CommandDescription
git branchList local branches (* marks current)
git branch <name>Create a new branch (stays on current branch)
git switch <branch> or git checkout <branch>Switch to another branch
git switch -c <branch>Create and switch to new branch
git merge <branch>Merge <branch> into current branch
git branch -d <branch>Delete a branch (safe: only if merged)
git branch -D <branch>Force delete a branch
 
# Example workflow
git switch -c feature/login     # create & switch
# ... work, commit
git switch main
git merge feature/login          # merge into main
git branch -d feature/login      # delete

3. Understanding HEAD and branches

HEAD is a reference to the current branch (or commit). When you switch branches, Git updates HEAD and changes the working directory to match that branch's tip.

4. Merge conflicts

When two branches modify the same part of a file, Git cannot automatically merge. You must resolve the conflict manually.

  1. Git marks conflicted files; edit them to keep the desired changes.
  2. Remove conflict markers (<<<<<<<, =======, >>>>>>>).
  3. Stage the resolved files: git add <file>.
  4. Complete the merge: git commit (Git will generate a merge commit message).
Use git status during conflict to see which files need resolution.

5. Remote branches

Remote branches are references to the state of branches in remote repositories (e.g., origin/main).

git push origin feature      # push local branch to remote
git fetch origin             # update remote-tracking branches
git pull origin main         = git fetch + git merge
git branch -r                # list remote branches
git checkout -b local-name origin/remote-name  # track remote branch

6. Branching strategies (workflows)

  • Git Flow: main, develop, feature branches, release branches, hotfixes.
  • GitHub Flow: main, feature branches, pull requests. Simple and popular.
  • Trunk‑based: short‑lived branches, frequent merges to main.

7. Practice exercises

1. Create a new branch called bugfix and switch to it. What command?

Answer
git switch -c bugfix

2. You are on main and want to merge feature into it. What commands?

Answer
git merge feature

3. How do you delete a local branch named old-feature after it has been merged?

Answer
git branch -d old-feature

4. After pushing your branch to remote, a colleague asks you to pull their changes. How do you update your local branch with the remote version?

Answer
git pull origin your-branch
