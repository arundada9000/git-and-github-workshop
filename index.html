GIT & GITHUB
workshop

beginner → practical collaboration · 3 days · 12h total
Git official GitHub Download Git GitHub Docs interactive branching .gitignore templates

📘 Course overview — This workshop introduces students to version control using Git and collaborative development using GitHub. By the end, participants will understand how developers track code changes, collaborate with teams, and manage projects using modern industry tools. Designed for absolute beginners.

duration
3 days · 4h/day
total learning
~12 hours
target
absolute beginners
DAY 1 · FUNDAMENTALS

Git local

Understand version control & core Git workflow locally.

📌 topics

  • Intro to VC – centralized vs distributed, Git vs GitHub
  • Install & config – username/email, commit identity
  • First repo – init, .git dir, tracking files
  • Core workflow – working dir, staging, commit
  • Commits – snapshot, hashes, messages
  • Undoing changes – restore, unstage, amend
  • .gitignore – ignore build files & deps
commands covered
git init git status git add git commit git log git log --oneline git diff git restore git restore --staged git commit --amend

also: git config (global user.name/email) – essential but not listed

exercises

  • Create a repository
  • Track files and commit changes
  • Modify files & observe Git history
  • Undo mistakes safely
DAY 2 · COLLABORATION

GitHub & teams

Connect Git to GitHub and collaborate on shared repos.

📌 topics

  • Intro GitHub – remote repos, public/private, structure
  • Connect local to remote – remote add, push, pull
  • Cloning – git clone
  • Branching – feature branches, branch, checkout, switch
  • Merging – fast-forward, merge commits
  • Merge conflicts – why & how to resolve
  • Pull requests – code review, merging PRs
commands covered
git remote git remote add origin git push git pull git clone git branch git checkout git switch git checkout -b git merge

exercises

  • Create branches
  • Push branches to GitHub
  • Create pull requests
  • Resolve merge conflicts
DAY 3 · PRO WORKFLOWS

advanced & open source

Real‑world dev workflows, rebasing, stashing, contribution simulation.

📌 topics

  • Dev workflows – feature branch, GitHub Flow
  • Rebasing vs merging – linear history
  • Stashing – git stash / pop
  • Advanced history – reset, revert, blame, show
  • Open source sim – fork, clone, branch, PR
  • Best practices – commit messages, secrets, structure
commands covered
git rebase git stash git stash pop git reset git revert git blame git show

simulation + exercises

  • Fork & clone a repo
  • Create feature branch & commit
  • Open a pull request
  • Practice rebase interactively

final project – collaboration simulation

groups will:

  • Create a repository
  • Use branches for features
  • Collaborate through pull requests
  • Resolve conflicts
  • Merge completed features

🔄 real PR flow · code review · merging

learning outcomes

  • Understand version control concepts
  • Use Git for tracking history
  • Create & manage repositories
  • Work with branches and merges
  • Collaborate using GitHub
  • Resolve merge conflicts
  • Contribute to team projects

target audience

  • Beginner programmers
  • Students new to version control
  • Anyone interested in collaborative software dev

tools used

Git GitHub CLI

best practices

  • Clear commit messages
  • Structure commits properly
  • Avoid committing secrets
  • Manage repos effectively
